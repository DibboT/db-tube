Discuss the scope of var, let, and const

variables declared with the var can not have block scope and they can be declared inside a {} block. It can be accessed outside of the block. var statemnet declares function-scoped or globally-scoped variables.
In ES6, let was introduced. let can be re assignable. It declares block-scoped local variables. let is much more updated than var. If a variable declares with let in a block it is available for use within that block.
const was also introduced in ES6 along with let. the variable declared with const can not be updated or re-declared. variable declared with const have bloc scope. const is block-sccoped.

Tell us the use cases of null and undefined

In Javascript, Null means an empty value. It is also a primitive type in Javascript. When we assign null as a value to any variable which indicates empty or blank. That variable has no value.
undefined is one of the primitive types of Javascript. If a value was not returned , a function returns undefined. When we do not pass an argument for a function parameter, the default value is taken as undefined.
